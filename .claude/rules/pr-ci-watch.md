# PR 作成後の CI 監視ルール

PR を作成またはプッシュした後は、CI が通るまで監視すること。

## 手順

1. `gh pr checks <PR番号>` で CI の状態を確認する
2. 全チェックが pass するまで定期的に確認する（pending のジョブがある場合は待つ）
3. fail したジョブがあれば、ログを確認して修正し、再プッシュする
4. 再プッシュ後も同様に監視を続ける

## 注意

- CI が全部通るまでユーザーに「完了」と報告しない
- `ruff format`, `ruff check`, `mypy` などの lint 系は手元でも事前に実行できる
- テスト失敗時はログから原因を特定し、コード修正 → テスト再実行 → プッシュのサイクルを回す
