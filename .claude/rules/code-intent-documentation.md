# コードの意図を記録するルール

## 原則: WHY を残す

コードには **なぜそれが存在するのか（WHY）** を記録する。WHAT はコード自体が語る。

## ルール

- **公開関数** (`export`): JSDoc で目的・背景・使用箇所を書く。削除判断の材料を含める（例: 「〜が不要になれば削除可能」）
- **非自明なロジック**: 「なぜこの方法か」「なぜこの順序か」をインラインコメントで書く
- **ワークアラウンド**: 理由と、いつ不要になるかを書く
- **型・インターフェース**: 「何を表現しているか」「ライフサイクル」を書く
- **定数**: マジックナンバーの由来を書く
- **防御的デフォルト値・非自明な初期化**: なぜここでこの値を設定するのか理由を書く

## 書かなくていいもの

- 自明なゲッター/セッター
- フレームワークの定型パターン
- 型名やパラメータ名から明らかなこと
